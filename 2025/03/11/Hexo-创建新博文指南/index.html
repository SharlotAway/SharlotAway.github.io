Hexo 创建新博文指南

2025/3/11 Introduciton

使用命令行新建文章

进入命令行（Win + R，输入 cmd，点击回车），找到 Blog 所在的目录，使用如下命令新建博文。

hexo new "新建文章"

就会创建一个名为新建文章.md的文件，命令行提示。

INFO  Validating config
INFO  Created: \Path\to\your\Blog\source\_posts\新建文章.md

此时可以看到在source/_posts 文件夹下多出一个新建文章.md，打开就可以使用标准 markdown进行编辑了。

调试和查看

在编辑好之后，使用hexo查看刚刚编辑好的博文。

hexo clean # 清理旧的文件
hexo server # 允许你在本地查看生成的网站，进行调试

如果调试成功，就使用hexo推送博文。在部署之前需要配置好根目录和主题目录下的 _config.yaml文件以确保可以正常生成网页。

hexo clean
hexo generate # 生成静态网站文件，将 Markdown 文件和其他资源转换为 HTML 文件。
hexo deploy # 将生成的静态文件部署到指定的远程服务器或托管平台（如 GitHub Pages）。

一些小问题

在 particlex 主题下，代码高亮和 hexo 的默认配置冲突，在根目录下的 _config.yaml 做出如下修改。

# 旧版本
highlight:
    enable: false
prismjs:
    enable: false
# 如果是 7.0.0+，去掉 highlight 和 prismjs
syntax_highlighter:

在主题目录的配置文件_config.yaml中使用进行代码高亮的配置。

# Highlight.js
# https://highlightjs.org
highlight:
    enable: true
    style: github
