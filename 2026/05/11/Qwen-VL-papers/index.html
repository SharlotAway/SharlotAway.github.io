Qwen-VL

主要特点

Qwen-VL 系列的特色包括：

  1. 支持多语言指令（英文、中文等）
  2. 支持任意图文交错数据（arbitrary interleaved image-text data）
  3. 通过高分辨率细粒度训练数据集获得细节理解能力

模型结构

视觉编码器：Openclip-ViT-bigG/14，输入图像需要缩放到指定分辨率

模态适配器（Position-aware Vision-Language Adapter）: 随机初始化的单层 Cross Attention，通过一些可学习的查询（Embeddings）学习查询来自视觉编码器的键（图像特征），并将其维度固定到 256.

绝对位置编码：在 Cross Attention 的键值对上增加二维绝对位置编码

绝对位置编码

对于一个在二维网格中坐标为 $(i,j)$ 的图像块，通过 $PE(\cdot)$ 给它编码位置信息，在两个维度分别得到一个 $D/2$ 维的位置编码，拼接得到最终的位置编码

$$ PE(i,j)=\mathrm{Concat}(PE_h(i), PE_w(j)) $$

输入输出

图像输入：在图像特征的开头和结尾添加<img>和</img>两个标记（token）表示图像特征。

边界框输入输出：通过将边界框位置归一化到 $[0,1000)$ 之后编码到文本中确定位置,因此不需要额外的位置词汇。

边界框串格式：<box>$(X_\text{topleft},Y_\text{topleft}),(X_\text{bottomrigh}, Y_\text{bottomright})$</box>，该位置相当于两个顶点在图中的千分位位置。

输出格式<ref>[边界框对应的目标]</ref> <box>[边界框坐标]</box>

数据示例

Assistant: the beautiful flowers for design.<eos>

# 生成边界框（Caption with Grounding）

# 描述部分

训练

第一阶段（预训练）：大规模、弱标签、网络爬虫集图文对，只训练图像和适配器。

第二阶段（多任务）：高质量、细粒度 VL 标注数据，更大的输入分辨率，混合的图文数据

第三阶段（监督微调）：指令微调，提升对话能力；从单图场景（Image Caption，LLM 自行生成的对话数据）扩展到多图理解场景。

优化方向

  1. 更多模态：语音、视频
  2. 增强模型规模、训练数据，提高分辨率、细节把握能力
  3. 提高多模态生成能力（图像、语音生成）

Qwen2-VL

主要特点

Qwen2-VL 扩展了许多 VL 能力，包括对话、视频理解、标注、OCR、长文本理解、数学、代码、实时对话、公式识别、函数调用、UI交互，主要创新点

  • 长视频理解（20 min+）及其相关功能（对话、问答）
  • 基于视觉、文本输入的设备操作的鲁棒性 Agent 功能
  • 在中英文基础上扩展了其他语言

模型架构

Qwen2-VL 保留了 Qwen-VL 的框架，选择了 675 M 参数量的 ViT 作为视觉编码器，选择 Qwen2 作为 LLM，几个关键更新包括任意动态分辨率（Naive Dynamic Resolution）、多模态旋转位置编码（M-RoPE）和统一的图像、视频理解。

Naive Dynamic Resolution

Qwen2-VL 支持任意分辨率的输入（但是通常设置了最小分辨率和最大分辨率）。

绝对位置编码 $\to$ 2D-RoPE

2D-RoPE 是一种原位变换，对于网格坐标为 $(i,j)$ 的图像块，特征为 $\mathbf{x}$，通过矩阵乘法将特征旋转得到编码后的特征

$$ \mathbf{x}_{2D\text{-}RoPE} = \begin{bmatrix} \mathbf{R}_{h}(i, \theta) & 0 \\ 0 & \mathbf{R}_{w}(j, \theta) \end{bmatrix} \mathbf{x} $$

$R_{h/w}(\mathrm{pos},\theta)$ 是 1D 旋转矩阵，分别对高、宽进行处理，$\theta$ 是预定义的频率基数。一般来说，对于分配了 $D$ 维的轴，它的每个维度 $(d,d+1)$ 都有一个特定频率；在其中，维度对的索引 $k\in[0,D/2)$ 对应的频率基是

$$ \theta_k=10000^{-2k/D} $$

由于多张图的尺寸可能不同，直接做 Stack 行不通，Padding 长度差距太大，Qwen2-VL 采用了 packing，将多个图直接拼接到一个序列中，并控制序列长度。同时，在 ViT 后面接了一个 MLP 层将相邻的 $2\times 2$ 词元压缩成一个。

RoPE $\to$ M-RoPE

处理文本的时候，是 1D-RoPE；处理图像的时候，是 2D-RoPE；处理视频的时候，是 3D-RoPE。当碰到多个模态组合的时候，是按照最高维度的对角线往后排列的，可以看下图的示意。

例如视频的三个维度分别是 $2,2,3$，那么在位置编码空间中一定要给这个模态分配一个 $3\times 3\times 3$ 的空间，然后后续的模态所有维度的位置编码索引将从 $4$ 开始

同一的图像/视频理解

Qwen2-VL 的训练数据是多模态混合的训练机制。

  • 视频采样频率为 2 frame/s，采用 3D 卷积处理视频输入；
  • 为和视频保持一致，将每张图像视为两个相同的帧
  • 调整每个视频帧的分辨率，限制总 token 数量在 16384 以内

训练

训练设置：

Flash Attention

Fused operators: LayerNorm, RMSNorm, Adam

数据格式：Dialogue、Grounding、Agent

  • 每个条交互的开头和结尾都用<|im_start|>和<|im_end|> 隔开；
  • 用<|vision_start|>,<|vision_end|>分割图像路径；
  • 用<|object_ref_start|> 和 <|object_ref_end|> 划分框内对应目标的表达，用<|box_start|> 和 <|box_end|> 划分边界框文本；

Dialogue 监督的包括对每条交互的定位（两个标记的位置）和模型输出内容；

Visual Grounding 监督是否生成边界框，并监督位置；

Visual Agent

Qwen2-VL 可以通过多步动作执行来完成 VL-Agent，在任务开始前，系统会为模型定义一组允许执行的动作（Permissible Actions）和用于函数调用的关键词模式 。Qwen2-VL 通过一个迭代循环来处理复杂任务 ：

  • 观察（Perception）：模型接收当前的视觉输入（如手机截图 Screenshot_1.jpg） 。
  • 推理与规划（Reasoning & Planning）：模型分析当前的屏幕内容并决定下一步。例如，在图中，模型识别出当前在邮件界面，若要找餐厅则需先返回主页 。
  • 执行动作（Action Execution）：模型输出指定的函数调用字符串，例如 *FUNCTION*: Home *ARGS*: {} 。

Qwen2.5-VL

主要特点

Qwen2.5-VL 主要替换了 Qwen2.5 backbone，对模型的各方面能力进行了提升包括

  • 文档解析能力（document parsing）
  • 各种格式的目标标定（objective grounding）
  • 超长视频理解和细粒度视频标定
  • 各种设备上的 Agent 能力

模型架构

LLM： 从 Qwen2 升级成了 Qwen2.5，为了提升多模态理解，将 1D RoPE 升级成饿了与真实时间对齐的 M-RoPE

Ablolute Time M-RoPE

Qwen2-VL 中的视频采样是每秒两帧，因此 M-RoPE 对视频的编码第一维度是帧的序号，而 Qwen2.5-VL 则将第一维度和绝对时间对齐，使用 timestamp 作为时间标志。

Vision Encoder： 沿用 ViT 结构，

  • 大多数层（只有 4 层使用全局注意力）使用窗口注意力（$8\times 8$ 由个图像块组成，小于这个尺度不进行 Padding）减小注意力开销；

  • 采用 2D-RoPE 捕捉位置信息，视频输入采用 3D-RoPE，将连续的两帧组合到一起；

  • 训练和推理都须将图像宽高缩放到28的整数倍，然后切成 $14\times 14$ 的图像块进行编码。

  • 为了和 LLM 更好配合，使用了 RMSNorm 和 SwiGLU

  • 从头训练 ViT，包括 CLIP 预训练，视觉语言对齐，端到端微调，采用了动态采样策略

MLP-based VL Merger： 在图像特征进入 LLM 前通过 Merger 进行压缩。首先将空间上相邻的四个图像块集合进行分组，拼接，然后通过一个 2 层 MLP 投影至文本空间（实际就是 LLaVA-NeXT 的投影，具体实现的时候也进行了 $2\times 2$ 压缩）

策略

Native Dynamic Resolution and Frame Rate 使模型更好感知真实物理量（空间位置，时间流逝）

  1. 不再缩放到固定尺寸，而是动态视觉分词并编码；
  2. 位置信息不再归一化，而是直接输入像素真实坐标
  3. 时间维度动态 FPS，根据语义变化强度决定采样强度，并且 M-RoPE id 对应真实时间

M-RoPE Aligned to Absolute Time

对时间戳编码，在不同 FPS，不同采样频率和不同视频速度下理解时间间隔与动作节奏。

下图展示的是对于一个八秒的视频，在不同采样频率下有不同的 M-RoPE ID 的标注，这里的 M-RoPE ID 是在 temperal merge 之后的（即一个 ID 代表两帧）

训练

预训练三个阶段以及对应的能力

后训练包括 SFT 和 DPO 两个阶段，先学会做任务，再符合人类偏好，后训练过程 ViT 保持冻结

  • SFT 采用 ChatML 格式，标记对话角色，嵌入视觉特征、文本指令和跨模态位置关系

  • 采用二阶段数据筛选

    • 第一阶段使用 Qwen2-VL-Instag 将 QA 划分为 8 个主领域和 30 个细粒度子域；
    • 第二阶段基于规则提出错误、不完整和有害的样本；然后基于 Qwen2.5-VL 系列训练的奖励模型在复杂度、相关性、正确性、清晰度和帮助性多个维度进行打分；
    • 拒绝采样：针对 CoT 的推理优化。（1）用中间版本对含 GT 的难题集生成多个预测答案以及推理步骤，（2）仅保留最终结果与 GT 完全匹配的样本；（3）对中间过程进行约束，包括代码切换，翻译不自然，长度过长火模式重复。

  • 使用 DPO 对齐人类意图：将图像-文本数据和纯文本数据利用偏好数据将模型行为对齐人类意图，每个样本仅处理一次，确保优化过程高效性。

Qwen3-VL

主要特点

发布了 Dense 和 MoE 版本的 Qwen3-VL，上下文长度达 256K 词元。在结构上优化了三个部分：

  • 交错旋转位置编码
  • 用于多层特征融合的 DeepStack
  • 显示视频时间戳

模型架构

LLM： 保留了 Qwen2.5-VL 的架构，用 Qwen3 作为骨架；

Vedio Timestamp

Qwen3-VL 在视频感知上将基于位置编码的隐式感知修改为了基于特殊词元的显示感知。M-RoPE 虽然可以增加时间编码，但是容易对长距离迷失，并且训练低效，因此直接插入一个格式化文本字符串（类似于 <3.0 seconds>）来显示标记时间。这样就不用推算而是直接知道了。

Vision Encoder： （1）使用 SigLIP-2 架构和预训练权重，并用动态分辨率的策略继续训练的视觉编码器；（2）2D-RoPE + 绝对位置编码插值（CoMP），将相对位置和绝对位置结合；

Interleaved MRoPE

传统的 RoPE 特征的维度切分成3块分给 $(t,h,w)$ 三个维度，类似于

$$ (t_1,t_2,\cdots,t_d,h_1,h_2,\cdots,h_d,w_1,w_2,\cdots,w_d) $$

因为 Transformer 的注意力权重作用在整个 $D$ 维，学习到的 “低频语义”（捕捉全局、长程关系的神经元）会被迫集中在向量的末尾。因此会发生类似无法处理长视频的现象。Qwen3-VL 在位置编码上将三个维度的旋转位置编码穿插排列在嵌入维度中，类似于

$$ (t_1,h_1,w_1,t_2,h_2,w_2,\cdots,t_d,h_d,w_d) $$

这样就可以在不同频段都处理三个轴的信息。

MLP-based VL Merger：（1） 2 层 MLP 对 $2\times 2$ 区域进行压缩和投影；（2）适配 DeepStack 机制的 Merger

DeepStack

Qwen3-VL 的 DeepStack 设计是从 Vision Encoder 的多个中间层提取不同层级的特征，然后通过专属 Merger 映射到多个 LLM 层中。这种思路的优势在于

  • 避免了将图像缩放多次，而是直接利用中间状态提取不同成绩信息；
