站点使用指南

2025/3/11

Hexo 站点维护说明

这份文档面向当前仓库的维护者，重点说明目录组织、个人信息修改方式，以及如何新增文章与分类页面。

文档组织结构

站点主要内容位于 source/ 目录下：

source/
├─ _posts/
│  ├─ blog-maintenance/          # 博客维护与 Hexo 使用说明
│  ├─ llm-foundations/           # 大模型基础
│  ├─ environment-setup/         # 环境配置
│  ├─ multimodal-paper-notes/    # 多模态论文阅读
│  └─ multimodal-practice/       # 多模态实验与工程实践
├─ about/                        # 关于页
├─ categories/                   # 分类页
├─ tags/                         # 标签页
└─ guide/                        # 本使用指南页面

文章文件采用“二级目录 + 英文 slug”的方式命名，例如：

source/_posts/multimodal-paper-notes/qwen-vl-paper-notes.md

如果文章启用了 post_asset_folder: true，则配套图片目录应与 Markdown 文件同名，例如：

source/_posts/multimodal-paper-notes/qwen-vl-paper-notes/

如何修改个人信息

个人信息主要分布在以下两个位置：

  1. 站点基础信息：根目录 _config.yml
  2. 主题展示信息：themes/particlex/_config.yml

常见修改项如下：

  • 站点标题：_config.yml 中的 title
  • 作者名称：_config.yml 中的 author
  • 站点地址：_config.yml 中的 url
  • 首页侧边栏简介：themes/particlex/_config.yml 中的 card.description
  • 头像、背景图、加载图：themes/particlex/_config.yml 中的 avatar、background、loading
  • 导航菜单：themes/particlex/_config.yml 中的 menu

“关于我”页面位于 source/about/index.md，其中可以直接修改：

  • 姓名、学校、研究方向
  • 联系方式
  • GitHub / Google Scholar 链接
  • 个人简介、获奖情况、论文列表

如何创建新的博文

建议先确定文章属于哪个分类目录，再创建文件。当前仓库更适合使用带 path 参数的方式新建文章：

hexo new post --path "multimodal-paper-notes/new-post-slug" "文章标题"

例如，在“多模态实践”分类下新建一篇文章：

hexo new post --path "multimodal-practice/qwen-visual-debugging" "Qwen 视觉调试记录"

执行后会生成：

source/_posts/multimodal-practice/qwen-visual-debugging.md
source/_posts/multimodal-practice/qwen-visual-debugging/

写作时建议补齐以下 front matter：

---
title: Qwen 视觉调试记录
date: 2026-05-27 10:00:00
tags:
  - 多模态
  - 实验实践
  - Qwen
categories:
  - 多模态实践
description: 简要说明文章内容。
---

如何创建新的分类

当前站点的“分类页”会自动根据文章 front matter 中的 categories 聚合生成，因此通常不需要额外手工创建分类页面。

新增分类时建议按下面流程进行：

  1. 在 source/_posts/ 下新建一个对应的英文目录名，例如 reading-notes/
  2. 新文章统一放进该目录
  3. 在文章 front matter 中把 categories 写成面向读者的中文分类名

例如，如果要新增“工具使用”分类，可以这样做：

source/_posts/tooling/

hexo new post --path "tooling/windows-terminal-tips" "Windows Terminal 使用技巧"

categories:
  - 工具使用

目录名负责仓库内部组织，categories 负责网站上的展示名称，两者可以不同，但建议长期保持稳定映射。

常用命令

本地预览：

npm run clean
npm run server

生成静态文件：

npm run clean
npm run build

部署到 GitHub Pages：

npm run deploy

主题与写作注意事项

  • 当前主题为 particlex，主题配置文件位于 themes/particlex/_config.yml
  • 当前仓库已经启用 post_asset_folder: true，文章图片请放在与文章同名的资源目录中
  • 根配置里已关闭 Hexo 默认代码高亮，改由主题中的 Highlight.js 配置负责渲染
  • 如果修改了导航、头像或首页信息，建议本地先执行 npm run server 检查显示效果
